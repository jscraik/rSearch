# Documentation QA Checklist

Use this checklist to review documentation updates before release.

## Structure and clarity
- [ ] Doc requirements section is present (audience, scope, non-scope, owner, cadence, approvals).
- [ ] Headings are informative and in logical order (no skipped levels).
- [ ] Table of contents matches section headings (if present).
- [ ] Paragraphs are short and scannable.

## Accuracy and completeness
- [ ] Examples match current CLI behavior and scripts.
- [ ] Links resolve to existing files or URLs.
- [ ] Troubleshooting steps include clear causes and fixes.
- [ ] Risks and assumptions are explicit where applicable.

## Accessibility and safety
- [ ] Links use descriptive text (no "click here").
- [ ] Inclusive language is used throughout.
- [ ] Security and privacy guidance is present where relevant.

## Verification
- [ ] Verification steps are included for operational docs.
- [ ] Evidence bundle is present and updated.
